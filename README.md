# Book Beacon — backend

*Every story finds its light.* A reading companion API: add books to your
shelf, keep a vault of words, quotes, and notes from what you read, and chat
with the keeper — an AI agent that recommends from your own pile.

Built to run **free, forever** — no paid tiers, no credit card.

## Zero-cost stack

| Piece | Service | Why |
|---|---|---|
| API hosting | Render free web service | No card, never expires (spins down when idle) |
| Database | Neon Postgres + pgvector | Permanent free tier, 0.5 GB, vector search built in |
| Book catalog | Google Books API | Free, no key needed |
| AI (optional) | Google Gemini free tier | 1,500 req/day; powers embeddings, image search, the keeper |

The core API (auth + search + pile + vault) runs with only a Neon URL and a
JWT secret. The AI features stay dormant until you add a Gemini key — so you
never pay.

## Run locally

```bash
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env        # set DATABASE_URL (Neon) and JWT_SECRET
alembic upgrade head        # create the schema + pgvector extension
uvicorn app.main:app --reload
```

Open http://localhost:8000/docs for Swagger. Click **Authorize**, register or
log in, and every protected route works from the browser.

## Get a free database (Neon)

1. Sign up at https://neon.tech (no card).
2. Create a project; copy the connection string.
3. It looks like `postgresql://user:pass@ep-xxx.aws.neon.tech/db?sslmode=require`.
   Change `postgresql://` to `postgresql+asyncpg://` and drop `?sslmode=require`
   (TLS is handled in code). Put the result in `DATABASE_URL`.

## Migrations (Alembic)

The schema is owned by Alembic, not auto-created. Common commands:

```bash
alembic upgrade head                      # apply all migrations
alembic revision --autogenerate -m "..."  # create a new migration after model changes
alembic downgrade -1                       # roll back one
```

The initial migration (`0001`) enables the `vector` extension and creates the
core tables. `0002` drops the retired `garden_plants` table and `0003` adds
`vault_entries`. `alembic/env.py` reuses the app's async engine, so it
inherits the same TLS handling as the app.

## Deploy free (Render)

1. Push this folder to GitHub.
2. On Render: **New > Blueprint**, point it at the repo (`render.yaml` included).
3. Set `DATABASE_URL` to your Neon URL. `JWT_SECRET` is auto-generated by Render.
   Optionally set `GEMINI_API_KEY` to switch on AI features.
4. Deploy. The start command runs `alembic upgrade head` before serving, so the
   schema stays in sync on every deploy.

## Endpoints

### Auth
| Method | Path | Description |
|---|---|---|
| POST | `/auth/register` | Create an account → returns a JWT (`{email, password, name?}`) |
| POST | `/auth/login` | Log in (form: `username`=email, `password`) → JWT |
| GET | `/auth/me` | Current user |

All shelf/vault routes require `Authorization: Bearer <token>`.

### Books
| Method | Path | Description |
|---|---|---|
| GET | `/search?q=` | Type-to-search Google Books |
| POST | `/search/manual` | Create a search result from manually entered details (no Google match); returns a `manual:` prefixed `google_id` to pass to `POST /shelf` or `POST /vault` |
| POST | `/search/identify` | Upload a cover photo → Gemini reads it → search (AI on) |
| POST | `/shelf` | Add a book to the pile (`{"google_id": "..."}`) |
| GET | `/shelf?status=` | List the pile (`to_read`/`reading`/`finished`) |
| PATCH | `/shelf/{id}` | Update status |
| DELETE | `/shelf/{id}` | Remove from the pile |

### Vault (words, quotes, notes)
| Method | Path | Description |
|---|---|---|
| POST | `/vault` | Save an entry (`{"book_id", "entry_type", "content", "page_number?"}`); `entry_type` is `word`/`quote`/`note` |
| GET | `/vault?book_id=&entry_type=` | List entries, optionally filtered by book and/or type |
| PATCH | `/vault/{id}` | Edit an entry's content or page number |
| DELETE | `/vault/{id}` | Remove an entry |

### The keeper (WebSocket)
Connect to `ws://localhost:8000/ws/clerk?token=<your-jwt>`, then send plain text
messages. The keeper replies as JSON `{role, content}` where `role` is `keeper`,
`system`, or `system: "typing"`. It's a Gemini agent with three tools:

- `search_books` — discover new titles in the wider catalog
- `recommend_from_my_pile` — semantic (pgvector) search over your unread books
- `get_my_stats` — your reading numbers

Requires `GEMINI_API_KEY`; without it the keeper politely reports it's off duty.

### Meta
| Method | Path | Description |
|---|---|---|
| GET | `/` | App name, AI-features status, docs link |
| GET | `/health` | Liveness check (`{"status": "ok"}`) |
| GET | `/docs` | Swagger UI |

## Enabling the AI features

Get a free key at https://aistudio.google.com/apikey and set `GEMINI_API_KEY`.
This turns on description embeddings (stored in pgvector on add), cover-image
search, and the keeper's recommendations.

## Project layout

```
app/
  main.py            FastAPI app + router wiring
  config.py          settings (pydantic-settings)
  database.py        async engine, session, Base
  models.py          SQLAlchemy models
  schemas.py         Pydantic request/response
  auth.py            password hashing + JWT
  deps.py            get_current_user (HTTP) + token resolver (WS)
  services/
    google_books.py  catalog search
    embeddings.py     Gemini embeddings + cover-image ID (optional)
    clerk.py          the keeper agent loop + tools
  routers/
    auth.py  search.py  shelf.py  vault.py  clerk.py
alembic/             migrations
render.yaml          one-click free deploy
```
